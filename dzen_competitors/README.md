# dzen_competitors

Анализ конкурентов в Дзене по нише. Анонимный парсинг через Playwright.

## Что делает

1. Разворачивает нишу в 10–30 поисковых запросов (из YAML-шаблонов).
2. Для каждого запроса скроллит `dzen.ru/search?query=…` и собирает **карточки статей** (`[data-card-type="card-article"]`): URL, заголовок, канал.
3. Для каждого найденного канала открывает его страницу, снимает **заголовок, описание, количество подписчиков** и список последних N статей.
4. Ранжирует каналы и кладёт CSV-отчёт в `data/`.

## Какие метрики используются

**Что Дзен отдаёт публично анонимному пользователю:**
- название канала, описание, подписчики;
- список статей канала (URL + заголовки);
- ссылки «канал → статья» через поиск.

**Что НЕ отдаёт** (проверено по реальному DOM в апреле 2026):
- просмотры, лайки, комментарии — скрыты от анонимных пользователей на страницах статей и в карточках;
- «дочитывания» — есть только в студии автора своих каналов;
- даты публикации в ленте канала — не выведены в DOM.

Из-за этого скоринг построен на том, что **реально доступно**:
- `subscribers` — подписчиков у канала.
- `niche_hits` — сколько разных запросов ниши попали в этот канал (сигнал релевантности теме).
- `articles_seen` — сколько статей канала всего собрано (из поиска + ленты канала).

**Формула скора:** `log10(subscribers+1) + 2 * log10(niche_hits+1)`

То есть большие тематические каналы с многократными попаданиями в нишу идут вверх; разовое попадание статьи мелкого канала — вниз.

## Запуск

```bash
# из корня проекта (один раз):
python3 -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt
playwright install chromium

# прогон:
cd dzen_competitors
python dzen_competitors.py run --niche "недвижимость"
```

Свой список модификаторов запроса:

```bash
cp queries_example.yaml queries.yaml   # отредактировать под нишу
python dzen_competitors.py run --niche "недвижимость" --queries queries.yaml
```

Для ниш, где шаблонные модификаторы («как ухаживать», «в домашних условиях»)
дают мусорные запросы, обязательно сделай свой YAML.

## Файлы

| Файл | Назначение |
|------|------------|
| `dzen_competitors.py` | CLI и оркестрация трёх фаз |
| `search_parser.py` | Сбор статей со страницы поиска |
| `channel_parser.py` | Обход страницы канала (подписчики, описание, ссылки) |
| `ranker.py` | Скоринг каналов + CSV |
| `storage.py` | SQLite (runs/channels/articles) |
| `browser.py` | Анонимный Chromium, вежливые паузы, скролл |
| `parsing_utils.py` | Счётчики «55,3 тыс» → 55300, slug канала, фильтр URL |
| `config.py` | .env + расширение ниши в запросы |
| `queries_example.yaml` | Шаблоны модификаторов |

## Настройка (.env)

Все параметры опциональны, см. `.env.example`. Главные:

- `MIN_DELAY` / `MAX_DELAY` — паузы между действиями (1–3 сек по умолчанию; поднимай, если словишь тротлинг).
- `MAX_ARTICLES_PER_QUERY` / `MAX_CHANNELS` / `MAX_ARTICLES_PER_CHANNEL` — лимиты прогона.
- `HEADLESS=false` — посмотреть, как всё ходит, при отладке.

## Что учти

- **Антибот.** Если вырастает процент пустых запросов или Дзен начинает показывать капчу, увеличь `MIN_DELAY`/`MAX_DELAY`, уменьши `MAX_ARTICLES_PER_QUERY`.
- **Вёрстка Дзена плавает.** Парсер привязан к `data-card-type="card-article"` и классам `search--card-*`/`channel--card-*`. Если найдёшь пустую выдачу после обновления Дзена — открой страницу руками и обнови селекторы в `search_parser.py` / `channel_parser.py`.
- **Персонализация.** Поисковая выдача Дзена слегка персонализирована даже анонимно (IP/язык/время). Для повторяемости используй одну и ту же сеть.

## Повторный прогон и диффы

Всё пишется в `data/dzen_competitors.sqlite` с уникальностью по URL. Повторный запуск по той же нише обновит метрики (подписчиков, попадания) и даст свежий CSV; историю прогонов видно в таблице `runs`. Диффы между прогонами можно считать SQL-запросом по `runs.started_at`.
